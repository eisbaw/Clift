## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Build a Lean 4 @[env_linter] for semantic proof quality checks that grep/Python cannot do. These require the Lean kernel and type system to evaluate.

LOCATION: tools/lint/SemanticLint.lean (or Clift/Lint/SemanticLint.lean)

WHY: Python audit catches textual anti-patterns but cannot:
- Determine if a postcondition is definitionally equal to True
- Check if a precondition is inhabited (has any satisfying state)
- Verify that satisfiedBy references the auto-generated L1 body (not just name matching)
- Detect that a spec constrains behavior (not just restates the definition)

CHECKS TO IMPLEMENT:

1. tautological_postcondition_linter:
   - For each FuncSpec definition, extract the post field
   - Try to prove (∀ r s, post r s) via decide/trivial/rfl
   - If provable: FAIL — postcondition is tautological
   - Known hits: rb_count_nodes_spec, rb_sum_spec, rb_count_above_spec, rb_count_at_or_below_spec

2. vacuous_precondition_linter:
   - For each FuncSpec, extract the pre field
   - Try to prove ¬(∃ s, pre s) via decide
   - If provable: FAIL — precondition is unsatisfiable (theorem is vacuously true)

3. satisfiedBy_target_linter:
   - For each theorem using satisfiedBy, extract the L1 body argument
   - Check: is it a declaration generated by clift (in the Module namespace)?
   - If not: WARN — proof may target a hand-written body

4. axiom_audit_linter:
   - For each theorem in Examples/, run #print axioms equivalent
   - Flag any theorem depending on sorryAx or Lean.trustCompiler
   - Report unexpected axiom dependencies

5. spec_strength_linter:
   - For each FuncSpec, check if postcondition references ret__val or heap state
   - If post only references reflexive equalities (x = x): WARN — likely too weak
   - If post doesn't mention ret__val at all: WARN — doesn't constrain return value

INTEGRATION:
- Run via: lake build Clift.Lint.SemanticLint (or import in a test file)
- #lint command runs all registered linters
- CI job runs after build passes

PRIOR TASKS TO REVIEW AND SUBSUME:

- TASK-0231 (Strengthen tautological FuncSpec postconditions) — detection part subsumed, strengthening still separate

- Meta.isProof, Meta.inferType for type inspection
- Meta.isDefEq for checking definitional equality with True
